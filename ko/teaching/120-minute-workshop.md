# 120분 워크숍

[HEAD Agent Core](../README.md) / [교육](README.md) / 120분 워크숍

## 결과

참가자는 전체 아키텍처를 가상의 오래 유지되는 결과에 적용합니다. 의사결정 권한, 컨텍스트 출처, 근거 게이트, 오래 유지되는 합의, 최소 도입 권고를 갖춘 경계가 정해진 작업 모델을 만듭니다.

## 준비

- 가능한 경우 다양한 직군을 섞어 3~5명씩 그룹을 구성합니다.
- 가상의 [도서관 키오스크 파일럿](../learn/08-operation/end-to-end-example.md) 또는 비공개 사실이 전혀 없는 동등하게 가상의 시나리오를 사용합니다.
- [정본 다이어그램](diagrams.md)과 [워크숍 질문](discussion-questions.md#워크숍-연습)을 엽니다.

## 그룹 워크시트

각 그룹은 다음만 쉬운 말로 작성합니다.

1. 사용자가 확정한 목표와 추론해서는 안 되는 중요한 결정 하나.
2. 일관된 다음 결과 하나, 그 책임자, 직접 완료 근거.
3. 해당 책임자에게 필요한 가장 작은 권위 있는 컨텍스트.
4. 이 결과가 이후 작업의 입력이 되기 전의 근거 게이트.
5. 전체 결과를 재개하는 데 필요한 세션 정체성과 전체 런의 사실. 진행 상황과 이력은 하위로 유지한다.
6. 시나리오에 맞는 가장 작은 도입 수준과 한계 하나.

## 진행자 점검

- 독립적으로 관찰 가능한 결과가 없는 단계 목록은 거절하고, 경계가 정해진 결과로 다시 구성합니다.
- 중요한 방향은 HEAD나 에이전트가 추론하게 두지 말고 사용자에게 돌려보냅니다.
- 상태 주장 대신 직접 근거를 요청합니다.
- 진행 상황과 이력은 검색 기록으로 유지합니다. 세션 정체성이나 전체 런 합의를 대체하게 두지 마세요.
- 프로젝트 소유 사실을 공유 Core 지침에 넣지 마세요.
- 과정이 `Historical record(역사적 기록)`으로 레이블하지 않은 이론적 언어는 해석으로 다룹니다.

## 되짚기

그룹 결과를 [가상의 엔드투엔드 예시](../learn/08-operation/end-to-end-example.md)와 비교하세요. 목표는 그 시나리오를 재현하거나 모든 구성 요소를 더하는 것이 아닙니다. 적절한 규모에서 같은 권위, 정본, 구성 요소 경계, 검증 논리를 적용하는 것입니다.
