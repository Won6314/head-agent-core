# HEAD 교육

[HEAD Agent Core](../README.md) / 교육 | [English](../../teaching/README.md)

이 경로는 학습 과정을 개발자, 프로덕트 매니저, 디자이너, AI 운영 담당자가 함께하는 기술 청중에게 제공 가능한 세션으로 바꿉니다. 과정이나 현재 구현 참고 자료를 대체하지 않으면서도, 같은 통제된 확장 논지를 가르칩니다.

## 경로 선택

| 가능한 시간 | 경로 | 참가자가 할 수 있게 되는 일 |
| --- | --- | --- |
| 20분 | [소개](20-minute-introduction.md) | 검증 경계와 사용자, HEAD, 에이전트의 역할을 식별한다. |
| 60분 | [핵심 강의](60-minute-core-lecture.md) | 오너십, 컨텍스트 거버넌스, 정본, 검증이 왜 함께 작동하는지 설명한다. |
| 120분 | [워크숍](120-minute-workshop.md) | 모델을 가상의 결과물에 적용하고, 대안을 평가하며, 최소 도입 경로를 선택한다. |

## 교육 자료

- [정본 다이어그램](diagrams.md): 재사용 가능한 관계 다이어그램의 단일 출처입니다. 경로에 복사하기보다 이 다이어그램에 링크하세요.
- [토론 질문](discussion-questions.md): 청중과 교육 목적별로 정리한 프롬프트입니다.
- [HEAD 학습](../learn/README.md): 전체 서술형 과정과 근거 레이블입니다.
- [참조](../README.md#참조): 현재 공유 계약과 구현 중심 페이지입니다.

## 진행 경계

이 경로를 지침으로 사용하되, 정책을 만들어도 된다는 권한으로 사용하지 마세요. 다음 구분을 명시적으로 유지하세요.

- 사용자는 방향과 중요한 결정을 확정합니다.
- HEAD는 작업 모델, 근거 선택, 통합, 일상적인 계획 수립을 책임집니다.
- 경계가 정해진 에이전트는 제공된 경계 안에서 로컬 실행을 책임지고, 직접 완료 근거를 반환합니다.
- 생성된 결과는 자신의 주장에 적합한 근거로 확인되기 전까지 초안입니다.

과정의 `Historical record(역사적 기록)`, `Operational observation(운영 관찰)`, `Generalized failure(일반화된 실패)`, `Related theory(관련 이론)` 근거 레이블은 교육 상황에서도 같은 의미를 유지합니다. 사후 이론을 원래의 설계 역사로 제시하거나, 공개 예시로 비공개 프로젝트 세부 사항을 추론하지 마세요.

## 진행 패턴

1. 학습 목표와 세션 경계를 말합니다.
2. 링크된 학습 페이지로 가르치고, [정본 다이어그램](diagrams.md)의 각 다이어그램을 사용합니다.
3. 참가자에게 관찰된 근거와 그럴듯한 결과를 구분하게 합니다.
4. 결정 책임자 한 명, 근거 게이트 하나, 경계가 정해진 다음 결과 하나를 말하게 하며 마무리합니다.

첫 세션에는 [20분 소개](20-minute-introduction.md)로 시작하세요. 완전한 개념 순서가 필요하면 [60분 핵심 강의](60-minute-core-lecture.md)를 사용하세요.
