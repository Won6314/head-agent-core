# 제어 평면으로서의 HEAD

[HEAD Agent Core](../../README.md) / [학습](../README.md) / [소유권](README.md) / 제어 평면으로서의 HEAD

## 학습 목표

근거 선택, 의존성 구성, 작업 배정, 정본 결론 산출에서 HEAD의 역할을 이해한다.

## 조정은 생산적인 작업이다

HEAD는 단순히 메시지를 전달하는 대신 전체 결과를 소유한다. 현재 상태와 의도한 결과를 세우고, 권위 있는 근거를 찾으며, 의존성과 사용자 의사결정 게이트를 해결하고, 위임이 도움이 되는지 선택하며, 국소 작업이 돌아온 뒤 구성을 검증한다.

```text
근거 -> 작업 모델 -> 경계가 정해진 배정 -> 결과 검증 -> 정본 결론
                    ^                              |
                    +-------- 근거에 의해 수정 ---+
```

이 제어 평면은 작업 가까이에 있어야 한다. 근거가 들어온 뒤에도 바뀔 수 없는 계획은 조정이 아니라 취약한 인계다.

## 실행은 하위 작업이 아니다

에이전트는 자신에게 경계가 정해진 결과의 실행을 소유한다. 위임이 지연, 모호성, 조정 비용을 더할 때는 HEAD도 직접 실행할 수 있다. 구분 기준은 지위가 아니라 소유권의 범위다.

## 사후적으로 연결한 이론

**관련 이론, 사후적:** 제어 평면과 실행 평면의 구분은 유용한 시스템 비유다. 관측 가능한 결과의 방향성 그래프로서 의존성 스케줄링도 마찬가지다. 이 개념들은 원래의 의식적 용어였다고 주장하지 않으면서 현재의 역할 분리를 설명한다.

## 흔한 오해

HEAD를 제어 평면이라 부른다고 해서 모든 기술적 선택을 중앙화해야 한다는 뜻은 아니다. 국소 방법을 중앙화하면 전문가에게 경계가 정해진 결과를 주는 이유가 사라진다.

## 요점

HEAD는 근거와 전체 결과를 조정하고, 에이전트는 그 안에서 책임 있는 국소 결과를 실행한다.
