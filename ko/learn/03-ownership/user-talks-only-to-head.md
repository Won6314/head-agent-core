# 사용자는 HEAD하고만 대화한다

[HEAD Agent Core](../../README.md) / [학습](../README.md) / [소유권](README.md) / 사용자는 HEAD하고만 대화한다

## 학습 목표

여러 에이전트가 작업에 기여하더라도 사용자가 하나의 대화 및 의사결정 표면을 갖는 이유를 이해한다.

## 하나의 표면, 하나의 책임 소유자

사용자는 방향, 절충, 중요한 결정을 HEAD에 전달한다. HEAD는 그 방향을 작업 모델에 반영하고, 유용할 때 경계가 정해진 실행을 위임하며, 통합된 결론을 돌려준다. 에이전트는 계획을 바꾸는 근거를 제시할 수 있지만, 전체 결과를 사용자와 독립적으로 협상하지는 않는다.

이는 익숙한 실패를 피한다. 서로 다른 에이전트가 의도의 서로 다른 조각을 받고, 각자 합리적으로 해석하여, 원래 결정을 다시 찾아내지 않고는 조정할 수 없는 결과를 내는 실패다.

```text
사용자 방향 -> HEAD의 공유 작업 모델 -> 경계가 정해진 에이전트 결과
                         ^                         |
                         +---- 근거와 질문
```

## 일반화된 실패

여러 에이전트는 서로 양립하지 않는 가정에 기대면서도 각각 질문에 설득력 있게 답할 수 있다. 문제는 에이전트가 유용한 추론을 할 수 없다는 데 있지 않다. 그럴듯한 답들 중에서 고르기 위해 필요한 의존성, 절충, 사용자 결정을 전체로 소유하는 에이전트가 없다는 데 있다.

설계 대응은 에이전트의 근거를 숨기는 것이 아니다. HEAD는 하나의 현재 작업 모델에 대해 최종 구성을 책임지게 하고, 중요한 선택은 사용자에게 에스컬레이션한다.

## 사후적으로 연결한 이론

**관련 이론, 사후적:** 이는 시스템 설계의 단일 책임 인터페이스 및 계층적 계획의 단일 조정 지점과 닮아 있다. 이 명칭들은 현재의 구성을 설명할 뿐, 문서화된 최초 근거로 제시되지는 않는다.

## 흔한 오해

사용자 대면 소유자가 한 명이라고 해서 HEAD가 모든 작업을 직접 실행해야 하는 것은 아니다. 이는 위임이 전체 결과에 대한 책임을 이전하지 않는다는 뜻이다.

## 요점

여러 기여자는 실행을 개선할 수 있다. 책임 있는 하나의 대화 소유자는 의도를 보존한다.
