# 최소 도입

[HEAD Agent Core](../../README.md) / [학습](../README.md) / [도입](README.md) / 최소 도입

## 학습 목표

완전한 운영 체제를 가져오지 않고 필요한 권한 경계를 보존하는 가장 작은 구성으로 시작합니다.

## 핵심 주장

최소 실행 가능 모델에는 공유 추론 계층, 프로젝트 소유 계층, 인간 소유 방향이 있습니다. 오래 유지되는 작업에는 세션 정본을 추가하고, 진정으로 경계가 정해진 결과에만 워커 경로를 추가합니다.

```text
사용자: 목표와 중요한 선택을 고정
        |
        v
Shared Core: 휴대 가능한 책임 및 추론 원칙
        +
프로젝트 계층: 지역 권위, 근거 색인, 제약
        |
        v
HEAD: 모델링하고, 실행 또는 위임하고, 검증하고, 결론을 냄
```

## 작게 시작하기

1. HEAD가 전체 결과를 소유하고, 권위 있는 컨텍스트를 의도적으로 조합하며, 응집된 책임을 보존하고, 결과를 검증한다고 말하는 압축된 Shared Core를 사용합니다.
2. 지역 권위, 제약, 진실 소스, 승인 경계를 정의하는 프로젝트 소유 지침을 만듭니다. 상세 근거를 모든 대화에 복사하는 대신 가리키는 작은 색인을 유지합니다.
3. 작은 작업에는 HEAD가 직접 작업하게 합니다. 조정, 중단, 복구, 인계에 유용할 때만 오래 유지되는 세션 정본을 기록합니다.
4. 오래 유지되는 작업에는 안정적인 세션 정체성을 전체 사용자-HEAD 합의와 분리합니다. 합의는 요구사항, 범위, 성공 조건, 작업 모델, 체크리스트, 현재 위치, 가정, 다음 행동, 근거 참조를 보유합니다. 요약은 검색에 도움이 될 수 있지만 합의를 대체할 수 없습니다.
5. 한 소유자가 명확한 결과를 처음부터 끝까지 맡을 수 있을 때만 선택적 워커 경로를 추가합니다. 그 소유자에게 가장 작은 완전한 컨텍스트를 주고 직접 결과 근거를 요구하며 HEAD가 검증된 결과를 통합하게 합니다.

## 아직 추가하지 않을 것

큰 명단, 전문 통합, 의무 템플릿, 작업 상태 데이터베이스로 시작하지 마세요. 이들은 이후 실제 문제를 풀 수 있지만 초기 작업에 필요하지 않은 유지보수와 권한 표면도 만듭니다.

## 설계 대응

공유 [Core](../../../head/README.md) (영문)는 휴대 가능한 행동을 정합니다. [Project Layer](../../../projects/README.md) (영문)는 지역 의미를 더합니다. 런 중심 절차는 [Shared Skills](../../../skills/README.md) (영문)를 통해 이용할 수 있으며, 조정 인터페이스와 재사용 역할은 선택적인 [MCP](../../../mcp/README.md) (영문) 및 [Agent](../../../agents/README.md) (영문) 계층입니다.

## 흔한 오해

“최소”는 “명세 부족”을 뜻하지 않습니다. 작은 시스템에도 명확한 결과, 권한 경계, 완료를 관찰할 방법이 필요합니다.

## 요점

인간 방향, 공유 원칙, 프로젝트 권위, 직접 검증으로 시작하세요. 작업이 요구할 때만 지속성과 위임을 추가하세요.
