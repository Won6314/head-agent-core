# 한계

[HEAD Agent Core](../../../README.md) (영문) / [Learn](../../../learn/README.md) (영문) / [도입](README.md) / 한계

## 학습 목표

모델이 개선할 수 있는 것과 여전히 불확실하거나 비용이 들거나 인간이 소유하는 것에 대한 명확한 기대를 갖고 모델을 도입합니다.

## 핵심 주장

HEAD는 작업을 더 검사 가능하고 복구 가능하게 할 수 있지만, 진실, 도구의 올바른 사용, 올바른 권한 배분, 좋은 인간 판단을 보장할 수는 없습니다.

| 한계 | 결과 |
| --- | --- |
| HEAD는 틀릴 수 있다 | 응집된 작업 모델도 불완전하거나 잘못된 근거에 놓일 수 있다. 변경 가능한 사실을 다시 확인하고 중요한 곳에서는 반박을 요청한다. |
| 위임에는 비용이 든다 | 브리핑, 대기, 검증, 통합은 직접 작업보다 느릴 수 있다. |
| 잘못된 권한 경계는 확산될 수 있다 | 나쁜 권한 경계는 워커와 도구 전반에 걸쳐 실수를 일관되게 전파할 수 있다. |
| 도구 이용 가능성이 적절한 사용을 보장하지 않는다 | 호출 가능 역량은 무시되거나, 오용되거나, 사용할 수 없거나, 실제 조건에 불충분할 수 있다. |
| 정본은 합의를 보존할 뿐 현실을 보존하지 않는다 | 오래 유지되는 기록은 범위 표류를 막는 데 도움이 되지만 새로운 외부 근거로 착각해서는 안 된다. |
| 독립 검토에는 경계가 있다 | 검토자는 오류를 놓치거나, 같은 사각지대를 공유하거나, 결과에 이의를 제기할 근거가 부족할 수 있다. |
| 사용자 방향은 여전히 필요하다 | 제품, 정책, 아키텍처, 비용, 위험, 작업 흐름, 공개 커뮤니케이션에 관한 중요한 선택은 프로세스 설계로 위임해 없앨 수 없다. |

## 설계 대응

직접 근거를 사용하고, 오래 유지되는 작업에는 사용자-HEAD 합의를 보존하며, 독립 검토가 중요한 결과를 실질적으로 바꿀 수 있을 때 추가합니다. 근거가 없거나 결정이 인간 경계를 넘으면 그럴듯한 발명으로 채우지 말고 불확실성을 말합니다.

## 관련 이론

이 모델은 경계가 정해진 실행을 조정하는 제어 평면을 닮았지만 이 비유에는 한계가 있습니다. 소프트웨어 아키텍처 용어가 LLM 시스템을 결정론적이거나 완전하거나 자율 통치되게 하지는 않습니다.
