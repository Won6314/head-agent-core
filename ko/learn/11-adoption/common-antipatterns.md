# 흔한 안티패턴

## 학습 목표

정돈되어 보이지만 책임, 근거 또는 적응성을 약화하는 추가 요소를 알아봅니다.

## 핵심 주장

대부분의 도입 실패는 메커니즘을 권위로 취급하는 데서 옵니다. 프로세스가 판단을, 컨텍스트가 근거를, 워커 결과가 사용자-HEAD 합의를 대체합니다.

| 안티패턴 | 실패 이유 | 더 나은 대응 |
| --- | --- | --- |
| 모든 작업에 의식 | 계획 아티팩트가 작은 결과를 바꾸지 않고 주의를 소모 | 프로세스를 조정, 복구, 위험에 맞춰 조절한다. |
| 컨텍스트 축적 | 양이 오래되고 충돌하며 무관한 자료를 들임 | 결정을 바꾸는 가장 작은 권위 집합을 검색한다. |
| 워커 군집 | 많은 부분 소유자가 통합 공백과 호환되지 않는 가정을 만듦 | 하나의 응집된 결과를 한 소유자에게 배정하고 독립 결과만 병렬화한다. |
| 숨은 권한 | 워커나 도구가 중요한 제품 또는 정책 선택을 조용히 함 | 결정을 드러내고 인간 소유자에게 돌린다. |
| 정본으로서의 요약 | 정보가 손실된 인계가 범위, 제약, 미완료 작업을 빠뜨릴 수 있음 | 안정적인 세션 정체성과 전체 합의를 요약 밖에서 권위 있게 유지한다. |
| 도구 우선 설계 | 이용 가능한 인터페이스를 정당화된 작업 흐름으로 착각 | 도구를 고르기 전에 결과와 안전 경계를 정의한다. |
| 증명으로서의 체크리스트 완료 | 완료한 행동도 요청한 동작을 놓칠 수 있음 | 결과의 직접 근거를 요구한다. |
| 프로젝트 지침을 Core에 복사 | 지역 가정이 거짓 보편 규칙이 됨 | 프로젝트 사실과 절차는 프로젝트 소유 계층에 둔다. |

## 일반화된 실패

팀은 느린 진행을 보고 워커를 추가하고, 조정하려고 요약을 추가하고, 요약 표류를 고치려고 더 많은 제어 메커니즘을 추가할 수 있습니다. 보이는 시스템은 커지지만 원래 결과는 더 보기 어려워집니다. 기본값으로 또 다른 메커니즘을 더하는 것이 아니라 권위 있는 합의를 복원하고 책임 중첩을 줄이며 관찰된 필요를 다루는 계층만 남기는 것이 교정 행동입니다.

## 흔한 오해

“자율성은 에이전트가 모든 것을 결정한다는 뜻이다.” 유용한 자율성은 경계가 정해진 결과 안에서의 기술적 자유입니다. 정책, 제품 방향 또는 권한을 발명하는 일은 포함하지 않습니다.

## 요점

인상적으로 보이는 프로세스보다 명시적 책임, 직접 근거, 가장 작은 적절한 메커니즘을 우선하세요.
