# 실패한 설계

[HEAD Agent Core](../../README.md) / [학습](../README.md) / [기원](README.md) / 실패한 설계

## 학습 목표

합리적으로 보였던 여러 메커니즘이 나중에 축소되거나 제거된 이유와 실패한 설계에도 유용한 국소 통찰이 남을 수 있는 이유를 이해한다.

## 핵심 주장

폐기된 메커니즘 대부분은 어리석지 않았다. 바로 앞에 놓인 실패를 해결했다. 국소적인 해결책을 영구적이고 보편적인 계층으로 취급했을 때 해로워졌다.

## 상주 에이전트

### 시도한 이유

상주 에이전트는 역할별 컨텍스트를 보존하고, 터미널에 계속 표시되며, 시작 비용 없이 서로 관련된 작업들을 연이어 수행할 수 있었다.

### 효과가 있었던 점

에이전트는 안정된 정체성을 가졌고 반복해서 호출할 수 있었다. 긴 작업 흐름을 모니터링하기가 더 쉬워 보였다.

### 실패한 점

상주 컨텍스트는 낡아졌고, 무관한 작업을 축적했으며, HEAD가 현재 파악한 내용과 달라졌다. 여러 에이전트를 계속 실행하는 일은 프로세스, 라우팅, 복구, 구성의 부담도 늘렸다.

### 현재 결정

HEAD는 연속성을 소유하므로 상주한다. 에이전트의 컨텍스트는 경계가 정해진 하나의 결과를 수행하는 동안만 유지되어야 하므로 에이전트는 일회성이다.

## 대규모 작업 패키지

### 시도한 이유

컨텍스트 파일, 지침 파일, 계획, 상태 JSON, 번호가 붙은 단계 파일은 작업을 명시적이고 복구 가능하게 만들었다. 여러 산출물에 걸쳐 반복적인 제작이 필요한 작업에서 특히 유용했다.

### 효과가 있었던 점

입력과 출력을 검사할 수 있었다. 대화 밖에 진행 상황을 기록할 수 있었다. 실패한 단계를 반복할 수 있었다.

### 실패한 점

패키지 구조 자체가 목표가 되었다. 작업에 더 단순한 표현이 필요한 경우에도 HEAD는 스키마를 충족하고, 단계를 추적하고, 포인터를 갱신하는 데 노력을 들였다. 에이전트는 실제 결과가 아니라 단계 완료를 최적화할 수 있었다.

### 현재 결정

오래 지속되는 작업에는 관찰 가능한 체크리스트가 있는 하나의 자유 형식 런을 사용한다. 일관된 결과가 별도의 소유권이나 복구로 이득을 얻을 때만 선택적 슬라이스로 세부 사항을 추가한다.

## 엄격한 명령 스키마

### 시도한 이유

짧고 모호한 명령은 예측할 수 없는 탐색을 낳았다. 검증 규칙은 더 긴 지침, 명시적 편집 패턴, 표준화된 요약을 강제했다.

### 효과가 있었던 점

명백히 불완전한 명령은 에이전트에게 도달하기 전에 거부되었다. 전달과 형식이 더 일관되어졌다.

### 실패한 점

구문적 완전성이 의미적 정확성을 보장하지는 않았다. 상세한 명령에도 HEAD의 잘못된 진단이 들어갈 수 있었다. 스키마가 엄격해질수록 에이전트가 실제 국소 해법을 발견할 능력을 억누를 수 있었다.

### 현재 결정

목적, 권한, 경계, 관찰 가능한 결과를 정의한다. 그 프로토콜을 따르는 것 자체가 결과의 일부일 때만 프로토콜을 지정한다.

## 생성된 복구 명령

### 시도한 이유

초기의 압축 복구는 적절한 파일을 다시 읽거나 현재 단계를 재구성하는 데 자주 실패했다. 모델이 상태를 다시 찾아내게 하는 것보다 독립적으로 완결된 주입 명령이 더 안전해 보였다.

### 효과가 있었던 점

생성된 명령이 정확하면 복구는 즉각적이고 결정론적으로 이루어졌다.

### 실패한 점

생성된 명령도 파생 요약이었다. 원래 범위를 빠뜨리거나 부분 완료를 잘못 분류하면 다음 세션은 그 오류를 지침으로 물려받았다.

### 현재 결정

복구 시 고정된 세션 정체성과 런에 담긴 전체 사용자-HEAD 합의를 불러온다. 생성된 진행 상황은 검색과 인계에 여전히 유용하지만 정본을 덮어쓸 수 없다.

## 활성 포인터와 대체 경로 선택

### 시도한 이유

런, 슬라이스, 스냅숏, 복구 기록이 여러 개일 때 자동 탐색을 이용하면 가장 관련성이 높은 항목을 선택할 수 있을 것으로 보였다.

### 효과가 있었던 점

사용자에게 파일을 지정해 달라고 요청하지 않고도 런타임이 작업을 재개할 때가 많았다.

### 실패한 점

선택 로직이 숨은 권위를 만들었다. 오래된 포인터, 최근 파일, 그럴듯한 스냅숏이 실제 합의를 몰래 밀어낼 수 있었다. 대체 경로가 늘수록 잘못된 작업을 복구하는 방식도 늘어났다.

### 현재 결정

정본 세션 파일에는 고정 경로를 사용한다. 고정 파일이 없다면 대체 파일을 추론하지 말고 생략한다.

## 많은 전문 에이전트

### 시도한 이유

역할 전문화는 더 나은 프롬프트, 더 명확한 전문성, 병렬 실행을 약속했다.

### 효과가 있었던 점

일부 결과는 전담 역할과 제한된 근거 범위에서 실제로 이득을 얻었다.

### 실패한 점

에이전트 정의가 안정적인 소유권 경계보다 빠르게 늘어났다. 몇몇 역할은 누적된 지침에서만 주로 차이가 났고, 그중 하나를 고르는 일 자체가 조정 작업이 되었다.

### 현재 결정

재사용 가능한 소수의 결과 소유자를 유지한다. 권한, 근거 범위, 소비 가능한 결과가 의미 있게 구별될 때만 전문가를 추가한다.

## 단순화 패턴

```text
관찰된 실패
    -> 국소 메커니즘
    -> 메커니즘에 정책과 상태가 축적됨
    -> 조정 비용이 드러남
    -> 오래 남을 원칙을 추출함
    -> 더 이상 비용만큼 가치가 없는 메커니즘을 제거함
```

## 흔한 오해

단순화는 구조 없는 단일 에이전트 채팅으로 돌아간다는 뜻이 아니다. 원래 문제가 사라져서가 아니라 소유권과 정본이 더 명확해졌기 때문에 현재 시스템은 더 작다.

## 핵심 정리

더 단순하게 표현할 수 없는 경계를 강제하는 동안에만 메커니즘을 유지해야 한다. HEAD의 역사는 추가의 역사인 만큼 삭제의 역사이기도 하다.
