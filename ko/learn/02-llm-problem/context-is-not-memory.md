# 컨텍스트는 기억이 아니다

[HEAD Agent Core (영문)](../../../README.md) / [학습 과정 (영문)](../../../learn/README.md) / [LLM 문제 모델](README.md) / 컨텍스트는 기억이 아니다

## 학습 목표

중요한 작업이 LLM이 대화 속에서 같은 의미를 계속 "기억하는" 것에 의존할 수 없는 이유를 설명하는 실용적 사고 모델을 세운다.

## 배우 비유

LLM 대화는 연속적으로 느껴진다. 어시스턴트는 이전 결정을 언급하고, 어조를 유지하며, 일어난 일을 기억하는 듯 보인다.

운영을 위해서는 조금 더 불편한 비유를 사용한다.

> 모델은 프로젝트에 대한 체험적 기억을 계속 소유하는 동료보다는, 대본을 받아 다음 장면을 연기하는 배우에 가깝다.

대본에는 대화, 시스템 지침, 검색한 문서, 도구 결과, 요약이 들어갈 수 있다. 모델은 지금 받은 내용을 해석한다. 핵심 제약이 빠져 있거나, 묻혀 있거나, 다른 내용과 충돌하거나, 잘못 요약되어 있으면 대화가 연속적으로 보이더라도 수행은 달라질 수 있다.

```text
이전 대화
        +
시스템 및 프로젝트 지침
        +
검색한 근거
        +
생성된 요약
        |
        v
모델에 제시된 현재 컨텍스트
        |
        v
다음 응답 또는 행동
```

## 이 비유가 도움이 되는 지점

이 비유는 운영상의 사실 네 가지를 부각한다.

### 존재 여부가 중요하다

프로젝트 어딘가에 저장된 사실도 모델이 그 사실을 받거나 검색하는 방법을 알지 못하면 현재 결과에 영향을 주지 않는다.

### 배치가 중요하다

권위를 갖는 요구 사항은 오래된 대화, 에이전트 보고, 최근 요약과 경쟁할 수 있다. 양이나 최신성만으로 모델이 의도한 권위의 위계를 추론한다고 보장할 수 없다.

### 압축은 의미를 바꾼다

긴 대화를 압축하면 세부 사항이 빠지고 구분이 무너질 수 있다. "한 부분이 완료되었다"가 "작업이 완료되었다"로 바뀔 수 있다. 해결되지 않은 가정이 완전히 사라질 수도 있다.

### 반복은 소유권이 아니다

모델은 결정을 바꿀 권한을 소유하지 않아도 그 결정을 반복할 수 있다. 결정 권한은 모델이 드러내는 자신감 밖에서 명시적으로 표현해야 한다.

## 이 비유가 들어맞지 않는 지점

배우 비유가 모든 모델 구현을 문자 그대로 설명하는 것은 아니다. 런타임 시스템은 토큰을 캐시하고, 대화 기록을 유지하고, 기억을 검색하거나, 한 번의 생성 안에서 숨은 상태를 사용할 수 있다. 모델은 이 비유가 암시하는 것보다 긴 컨텍스트를 더 잘 추론할 수도 있다.

실용적 요점은 더 좁다. 시스템이 어떤 메커니즘을 프로젝트에서 지속적으로 권위를 갖는 것으로 의도적으로 지정하지 않는 한, 그 메커니즘을 그런 권위를 가진 것으로 취급해서는 안 된다.

## 설계 대응

HEAD는 일상적인 채팅에서 뒤섞이기 쉬운 몇 가지를 분리한다.

| 필요 | 외부 표현 |
| --- | --- |
| 안정적인 행동 | 공유 코어와 프로젝트 코어 |
| 프로젝트에서 권위를 갖는 정보의 위치 | 프로젝트 정보 인덱스 |
| 이 세션의 주제 | 세션 컨텍스트 |
| 사용자와 HEAD가 달성하기로 합의한 것 | 런 정본 |
| 최근에 일어난 일 | 검색 전용 진행 상황과 이력 |
| 지금 참인 것 | 다시 확인한 1차 근거 |

모델은 여전히 대본을 읽는다. 시스템은 대본의 권위, 관련성, 복구 가능성을 개선한다.

## 일반화된 실패

긴 작업이 여러 차례의 대화 동안 이어졌다. 압축 뒤의 요약에는 최근 출력이 남았지만 원래 성공 조건은 빠졌다. 어시스턴트는 유창하게 작업을 이어 가고, 최신 산출물을 검토한 뒤 완료를 선언했다. 대화는 일관되게 느껴졌지만 작업은 더 이상 같은 작업이 아니었다.

해결책은 더 나은 알림 문구가 아니었다. 원래 합의를 안정적인 외부 기록에 보관하고 압축 뒤에 그 기록을 직접 불러오는 것이었다.

## 흔한 오해

"컨텍스트는 기억이 아니다"라는 말은 모델에 연속성이 없다거나 모든 프롬프트에서 모든 것을 반복해야 한다는 뜻이 아니다. 연속성이란 정본 기록, 의도적인 검색, 명시적 권위로 구성해야 하는 시스템 속성이라는 뜻이다.

## 핵심 정리

모델이 기억하는지를 묻지 말라. 지금 어떤 출처를 읽는지, 어떤 출처가 권위를 갖는지, 현재 컨텍스트가 사용자의 실제 합의를 보존하는지를 물어라.
