# HEAD 학습

[HEAD Agent Core](../README.md) / 학습 | [English](../../learn/README.md)

이 과정은 HEAD Agent Core가 존재하는 이유, 운영 중의 실패를 거쳐 설계가 어떻게 바뀌었는지, 그리고 그 구성 요소가 LLM과 함께 작업하는 통제된 방식을 어떻게 이루는지를 설명합니다.

이 과정은 하나의 구조화되지 않은 대화로 다루기에는 너무 길거나, 결과의 영향이 크거나, 컨텍스트가 많은 작업을 조정해야 하는 개발자, 프로덕트 매니저, 디자이너, AI 운영자를 위해 작성되었습니다.

## 과정 논지

HEAD는 자율적인 스웜을 만들려는 시도가 아닙니다. 통제된 확장을 위한 소유권 모델입니다.

```text
사용자가 방향과 중요한 결정을 확정
                    |
                    v
HEAD가 작업 모델을 만들고 전체 결과를 보존
                    |
                    v
경계가 정해진 하나의 에이전트가 일관된 하나의 결과를 소유
                    |
                    v
결과가 더 확장되기 전에 HEAD가 검증하고 통합
```

이 시스템은 네 가지 주장에 기반합니다.

1. LLM은 명확한 한 단계를 잘 구체화할 수 있지만, 확장이 후속 단계로 계속되면 검토되지 않은 출력은 불안정한 입력이 됩니다.
2. 컨텍스트 품질은 최대 분량이 아니라 권위, 관련성, 시점, 소유권에 달려 있습니다.
3. 사용자의 문제, 목표, 범위, 성공 조건은 손실이 있는 모델 요약 밖에서 정본으로 남아야 합니다.
4. 이식 가능한 행동은 생성적 원칙에서 나와야 하며, 상세 절차와 프로젝트 사실은 관련이 있을 때만 로드해야 합니다.

## 근거 레이블

이 과정은 네 종류의 주장을 구분합니다.

| 레이블 | 의미 |
| --- | --- |
| 역사적 기록 | 저장소 이력 또는 보관된 시스템 문서가 뒷받침합니다. |
| 운영 관찰 | 시스템을 실행하면서 반복적으로 관찰했지만 보편 법칙이라고 주장하지는 않습니다. |
| 일반화된 실패 | 프로젝트 식별 정보와 내부 세부 사항을 제거한 공개에 안전한 재구성입니다. |
| 관련 이론 | 설계를 설명하는 데 도움이 되는 확립된 아이디어이며, 때로는 회고적으로 식별됩니다. |

이 구분은 중요합니다. 현재 아키텍처는 필연적인 것이 아니었으며, 유용한 이론은 이력을 다시 쓰는 대신 근거를 명확히 해야 합니다.
