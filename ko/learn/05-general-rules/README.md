# 일반 규칙: 끝없는 금지 목록 대신 원칙

[HEAD Agent Core (영문)](../../../README.md) / [학습 과정 (영문)](../../../learn/README.md) / 일반 규칙

## 학습 목표

이식 가능한 Core 지침이 간결하고 생성적인 원칙을 선호하는 이유와, 상세한 조건부 절차가 필요 시 불러오는 Skills에 속하는 이유를 이해한다.

## 핵심 주장

금지 목록은 과거의 실수를 명명하며 자란다. 생성적 규칙은 명시적 권한 경계를 보존하면서 새 상황을 추론할 방법을 소유자에게 준다.

```mermaid
flowchart LR
    F[관찰된 실패] --> D[금지 추가]
    D --> X[더 많은 예외와 중복]
    F --> P[오래 유지되는 원칙 추출]
    P --> J[새 사례 추론]
    J --> V[근거와 대조해 검증]
```

## 장 구성

1. [금지 목록이 끝없이 늘어나는 이유](why-deny-lists-grow-forever.md)는 유지보수 함정을 식별한다.
2. [생성적 규칙](generative-rules.md)은 국소 추론을 안내하는 원칙을 정의한다.
3. [Core와 절차](core-vs-procedure.md)는 항상 로드되는 원칙과 조건부 Skills를 구분한다.
4. [에이전트 자율성 보존](preserving-agent-autonomy.md)은 경계가 정해진 결과가 정책 발명 없이 기술적 판단을 허용하는 방식을 보인다.
5. [HEAD Core 발전](evolving-head-core.md)은 지시문 본문이 아니라 설계 근거 수준에서 역사적 단순화를 설명한다.

## 이 장의 범위

이 장은 현재 공유 Core 및 계획 실무가 뒷받침하는 설계 전환을 설명한다. 지시문 본문을 재현하거나 작은 원칙 집합이 모든 예외 사례를 자동으로 해결한다고 주장하지 않는다. 프로젝트 정책, 안전 경계, 상세 절차는 필요한 곳에서 명시적으로 남는다.
