# 에이전트 자율성 보존

[HEAD Agent Core (영문)](../../../README.md) / [학습 과정 (영문)](../../../learn/README.md) / [일반 규칙](README.md) / 에이전트 자율성 보존

## 학습 목표

명확한 결과와 권한 경계가 에이전트가 정책이나 방향을 독립적으로 재정의하지 않고 기술적 판단을 쓰게 하는 방식을 본다.

## 자율성은 경계가 정해진 판단이다

모든 국소 선택이 미리 처방되면 에이전트는 유용하지 않다. 대상을 살피고, 적절한 방법을 고르고, 결과를 시험하며, 잘못된 전제를 보고할 여지가 필요하다. 이 여지는 HEAD가 이미 목적, 권한 경계, 완료 근거를 세웠기 때문에 존재한다.

```text
고정된 목적 + 관련 근거 + 권한 경계
                         -> 국소 기술 판단
                         -> 직접 근거
                         -> HEAD 통합
```

에이전트가 결과, 정책, 위험, 다른 중요한 결정을 바꾸는 선택을 만나면 자율성은 그 경계에서 끝난다. 에이전트는 근거를 제공하고, HEAD가 문제를 해결하거나 사용자에게 돌려준다.

## 미시 관리보다 유용한 이유

국소 경로를 처방하면 잘못된 진단을 보존할 수 있다. 무제한 탐색을 허용하면 검토되지 않은 범위가 생길 수 있다. 경계가 정해진 결과는 중간 경로를 만든다. 수용 대상은 안정적이되 기술 경로는 직접 근거에 맞춰 적응할 수 있다.

## 흔한 오해

자율성은 에이전트가 작업을 조용히 확장해도 된다는 뜻이 아니다. 배정된 결과를 어떻게 충족할지 소유한다는 뜻이며, 막히는 모호성을 드러낼 책임도 포함한다.

## 요점

기술적 자율성은 결과, 근거, 권한 경계가 모두 명시적일 때만 신뢰할 수 있다.
