# Core와 절차

[HEAD Agent Core (영문)](../../../README.md) / [학습 과정 (영문)](../../../learn/README.md) / [일반 규칙](README.md) / Core와 절차

## 학습 목표

공유 Core에 속하는 이식 가능한 원칙과 Skills에 속하는 상세 조건부 절차를 구분한다.

## 서로 다른 작업에는 서로 다른 전달이 필요하다

| 계층 | 담는 것 | 사용하는 때 |
| --- | --- | --- |
| 공유 Core | 안정적인 소유권 및 추론 원칙 | 모든 HEAD 결정에 같은 기반이 필요할 때 |
| 프로젝트 계층 | 프로젝트 소유 사실, 정책, 권위 포인터 | 작업이 그 프로젝트의 현실에 의존할 때 |
| Skill | 상세 조건부 절차와 도메인 사용 지식 | 특정 작업, 도구, 상황이 이를 요구할 때 |

항상 로드되는 지침은 작고 이식 가능해야 한다. 상세 절차는 특정 조건에 신뢰할 수 있는 순서를 줄 때 가치 있지만, 모든 절차를 모든 결정에 로드하면 Core는 이해하기 어렵고 오용하기 쉬워진다.

## 배치 검사

지침이 프로젝트를 가로질러 참인지, 보통의 추론에 필요한지 묻는다. 그렇다면 Core에 속할 수 있다. 도구, 환경, 특별한 위험, 작업 형태에 의존한다면 조건부로 두고 관련 있을 때 검색한다. 특정 프로젝트가 무엇을 할 수 있는지 정의한다면 소유자는 공유 Core가 아니라 프로젝트 계층이다.

## 흔한 오해

Core와 Skill은 품질 위계가 아니다. Skill은 특정 작업에 더 상세하고 더 중요할 수 있다. 차이는 범위, 소유권, 로드 시점이다.

## 요점

안정적인 원칙은 어디서나 로드하고, 상세 절차와 프로젝트 사실은 필요한 의사결정 경계에서만 검색한다.
