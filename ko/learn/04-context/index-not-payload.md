# 본문이 아닌 색인

[HEAD Agent Core (영문)](../../../README.md) / [학습 과정 (영문)](../../../learn/README.md) / [컨텍스트](README.md) / 본문이 아닌 색인

## 학습 목표

권위를 모든 프롬프트에 복사하지 않고 찾기 위해 색인을 사용한다.

## 색인은 지도이지 대체물이 아니다

프로젝트 색인은 어떤 종류의 사실, 정책, 기능을 어디에서 소유하는지 식별한다. 이는 HEAD에 검색 경로를 제공한다. 색인이 이를 이름으로 언급한다는 이유만으로 전체 정책, 근거 기록, 또는 진실의 출처가 되지는 않는다.

```mermaid
flowchart LR
    I[프로젝트 색인\n권위가 있는 곳] --> H[HEAD]
    H --> S[정본 출처]
    S --> J[현재 결과를 위한 판단]
```

이 구분은 복사된 요약이 요약 대상 출처보다 조용히 더 신뢰받는 것을 막는다. 또한 정본 문서가 각 소유자 아래에서 바뀌는 동안 색인을 간결하게 유지한다.

## 설계 대응

권위가 어디에 있는지 보여 주는 지도를 지속적으로 관리하고, 현재 질문에 답하는 출처를 검색한다. 거부된 대안은 문서 본문을 색인에 넣거나 프롬프트마다 중복하는 것이다. 중복은 표류를 만든다. 독자는 어느 것이 적용되는지 모른 채 두 버전을 볼 수 있다.

## 사후적으로 연결한 이론

**관련 이론, 사후적:** 이는 간접화와 진실의 출처 설계에 비유할 수 있다. 이 매핑은 설계의 원래 어휘에 관한 주장이 아니라 설명이다.

## 흔한 오해

색인은 출처가 최신이고 완전하며 적용 가능하다는 보장이 아니다. HEAD는 여전히 검색 여부를 판단하고 행동 전에 바뀔 수 있는 사실을 검증한다.

## 요점

작업 집합에는 권위로 가는 경로를 두고, 권위 자체는 소유된 정본 위치에 둔다.
