# 취약한 진행 상황과 이력

[HEAD Agent Core](../../README.md) / [학습](../README.md) / [정본](README.md) / 취약한 진행 상황과 이력

## 학습 목표

진행 상황과 이력을 사용자-HEAD 합의와 혼동하지 않고 유용한 검색 기록으로 사용합니다.

## 유용하지만 권위는 아니다

압축은 진행 상황 기록을 갱신하고 완료된 작업 이력을 추가할 수 있습니다. 이 기록은 HEAD가 이전 활동을 찾고, 어떤 근거를 다시 열지 결정하거나, 인계를 이해하는 데 도움이 될 수 있습니다. 이들은 LLM이 관리하며 불완전하거나 오래되었거나 이전의 잘못된 해석에 의해 형성되었을 수 있습니다.

| 기록 | 적절한 사용 | 허용되지 않는 역할 |
| --- | --- | --- |
| 진행 상황 | 가능한 현재 상태나 인계 찾기 | 범위를 다시 정의하거나 합의된 결과가 완료되었다고 선언하기 |
| 이력 | 과거 작업과 근거 후보 찾기 | 회고적 설명이 권위 있다고 증명하기 |
| 런 | 전체 결과 재개 및 판단 | 주장을 확인해야 할 때 상세 근거를 대체하기 |

```mermaid
flowchart LR
    P[진행 상황 또는 이력] --> R[후보 근거 검색]
    R --> C[컨텍스트와 런 확인]
    C --> J[HEAD 판단]
    P -. 판단보다 우선할 수 없음 .-> J
```

## 관찰된 이력과 해석

이력은 수행되었다고 말하는 내용을 기록합니다. 왜 수행되었는지, 원래 성공 조건을 충족했는지 또는 완료 주장이 이후 근거를 거쳐 유지되었는지는 자동으로 확립하지 않습니다. 이는 합의와 관련 근거에 비추어 확인해야 하는 회고적 판단입니다.

## 거부한 대안

진행 상황이나 이력을 자동 주입하면 취약한 표현이 복구가 시작되는 순간 불균형적으로 영향력을 갖게 됩니다. 현재 계약은 이 기록을 보존해 의도적으로 검색할 수 있게 두며, 복구 권위로 제시하지 않습니다.

## 요점

이력은 근거로 되돌아가게 할 수 있습니다. 그 근거를 해석하는 데 쓰는 합의를 대체할 수는 없습니다.
