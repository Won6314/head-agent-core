# 컨텍스트와 런

[HEAD Agent Core](../../README.md) / [학습](../README.md) / [정본](README.md) / 컨텍스트와 런

## 학습 목표

안정적인 세션 정체성과 전체 작업 합의를 구분하고, 두 가지를 함께 복구하는 이유를 이해합니다.

## 서로 다른 두 역할

복구 계약은 정체성과 작업 합의가 서로 다른 이유로 바뀌기 때문에 두 개의 분리된 파일을 사용합니다.

| 파일 역할 | 포함 내용 | 변경 경계 |
| --- | --- | --- |
| 세션 컨텍스트 | 세션 주제와 안정적인 제약 | 세션 자체가 바뀔 때만 변경됩니다. |
| 런 | 전체 사용자-HEAD 합의와 필수 체크리스트 | 합의는 안정적으로 유지되며, HEAD는 작업이 진전됨에 따라 현재 상황과 체크리스트를 갱신합니다. |

```text
세션 정체성: 이 세션을 규정하는 오래 지속되는 상황은 무엇인가?
        +
전체 런: 사용자와 HEAD는 어떤 결과를 제공하기로 합의했는가?
        =
복구 기반: HEAD는 무엇을, 어떤 제약 아래 계속해야 하는가?
```

## 한 파일로는 충분하지 않은 이유

두 파일을 합치면 변화하는 활동 기록이 세션의 안정적인 주제를 다시 정의하게 하거나, 안정적인 주제 설명이 작업 모델을 대신하게 만들 수 있습니다. 역할을 분리하면 복구 중 두 질문 모두가 보이게 됩니다.

슬라이스 브리프와 근거는 실행 세부 사항을 더할 수 있지만, 필요할 때 명시적으로 검색합니다. 이들은 상위 합의를 대체하거나 좁히지 않습니다.

## 흔한 오해

런은 경직된 템플릿이 아닙니다. 제목은 달라질 수 있습니다. 중요한 것은 완전한 합의, 체크리스트, 현재 상황, 미확인 사항, 다음 조치 및 근거 경로를 보존하는 것입니다.

## 요점

컨텍스트는 HEAD가 어떤 오래 지속되는 세션에 있는지 말합니다. 런은 HEAD가 어떤 전체 결과에 계속 책임이 있는지 말합니다.
