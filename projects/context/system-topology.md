## Why It Exists

System work depends on relationships among repositories, runtimes, hooks, services, data flows, and ownership boundaries. A queryable topology makes impact visible without placing the entire system graph in the default prompt.

## Boundary

Topology is a project-maintained map. Source code and live configuration remain the primary evidence when the map and implementation differ.
