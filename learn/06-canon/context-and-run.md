# Context And Run

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [Canon](README.md) / Context And Run

## Learning Objective

Distinguish stable session identity from the full work agreement, and understand why both are recovered together.

## Two Different Jobs

The recovery contract uses two separate files because identity and work agreement change for different reasons.

| File role | Contains | Change boundary |
| --- | --- | --- |
| Session context | The session topic and stable constraints | Changes only when the session itself changes. |
| Run | The full user-HEAD agreement and mandatory checklist | The agreement remains stable; HEAD updates current situation and checklists as work advances. |

```text
session identity: What enduring situation is this?
        +
full run: What outcome did the user and HEAD agree to deliver?
        =
recovery basis: What must HEAD continue, and under which constraints?
```

## Why One File Is Not Enough

Combining the two encourages a changing activity record to redefine the session's stable topic, or a stable topic description to stand in for the work model. Separating their roles keeps both questions visible during recovery.

Slice briefs and evidence may add execution detail, but they are retrieved explicitly when needed. They do not replace or narrow the parent agreement.

## Common Misunderstanding

The run is not a rigid template. Its headings may vary. What matters is that it preserves the complete agreement, its checklist, current situation, unknowns, next action, and evidence routes.

## Takeaway

Context says which enduring session HEAD is in. The run says what whole outcome HEAD remains responsible for.
