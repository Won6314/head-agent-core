# Generative Rules

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [General Rules](README.md) / Generative Rules

## Learning Objective

Understand what makes a rule generative and how it supports correct local action without authorizing invention.

## A Rule That Can Produce Judgment

A generative rule states the intended reasoning and boundary, not merely a remembered forbidden move. It helps an owner answer: what outcome am I responsible for, which authority controls this choice, what evidence matters, and when must I stop or escalate?

| Rule shape | What it supports |
| --- | --- |
| Preserve the user-approved outcome | Prevents local optimization from replacing direction |
| Use authoritative evidence for consequential claims | Prevents confidence from becoming proof |
| Give one owner a bounded result | Enables accountable technical judgment |
| Escalate choices outside the supplied authority | Prevents policy invention |

These are examples of reasoning shapes, not a replacement for a project's actual policies.

## Generation Still Needs Verification

A principle does not make an answer true. It gives a defensible way to choose a next action when the exact case was not enumerated. The result still requires evidence appropriate to its claim before it becomes trusted input to later work.

## Common Misunderstanding

Generative does not mean autonomous in every dimension. Technical method can be open within a bounded outcome while product, policy, risk, and other material choices remain outside the worker's authority.

## Takeaway

The best portable rule teaches how to reason inside a boundary and how to recognize the boundary.
