# Preserving Agent Autonomy

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [General Rules](README.md) / Preserving Agent Autonomy

## Learning Objective

See how clear outcomes and authority boundaries allow a worker to use technical judgment without independently redefining policy or direction.

## Autonomy Is Bounded Judgment

A worker is not useful when every local choice is prescribed in advance. It needs room to inspect the target, choose an appropriate method, test the result, and report a faulty premise. That room exists because HEAD has already established the purpose, authority boundary, and completion evidence.

```text
locked purpose + relevant evidence + authority boundary
                         -> local technical judgment
                         -> direct evidence
                         -> HEAD integration
```

If a worker encounters a choice that changes the outcome, policy, risk, or another material decision, autonomy ends at that boundary. The worker supplies evidence; HEAD resolves the issue or returns it to the user.

## Why This Is More Useful Than Micromanagement

Prescribing a local path can preserve a wrong diagnosis. Allowing unlimited exploration can create unreviewed scope. A bounded outcome creates a middle path: the acceptance target is stable while the technical path may adapt to direct evidence.

## Common Misunderstanding

Autonomy does not mean a worker may silently expand its task. It means the worker owns how to fulfill the assigned result, including the responsibility to expose a blocking ambiguity.

## Takeaway

Technical autonomy is credible only when the outcome, evidence, and authority boundary are all explicit.
