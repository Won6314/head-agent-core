# Composing Context

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [Operation](README.md) / Composing Context

## Learning Objective

Assemble the smallest authoritative working set that can change the current result.

## Core Claim

Context is selected by authority, relevance, timing, and ownership. An index can direct retrieval without becoming a payload dump; each owner receives only the information needed to make its decisions.

![HEAD context composition: portable principles, project rules, and the run canon inform HEAD, which retrieves relevant evidence for a bounded owner brief.](../../assets/diagrams/head-context.svg)

## Design Response

HEAD uses broad enough context to judge the whole outcome, then cites the selected evidence in a bounded brief. The brief includes purpose, locked decisions, boundary, relevant sources, and required evidence of completion. It omits broad history that does not change the result.

## Rejected Alternative

Loading everything looks safe because nothing is deliberately excluded. It instead introduces stale rules, conflicting facts, and hidden authority. A long context is not proof that the right source was used.

## Public References

The [project context index model](../../projects/context/README.md) explains indexed retrieval. The [shared Core](../../head/README.md) states the portable reasoning boundary. Neither page substitutes for project-owned authority.

## Takeaway

Retrieve context because it changes a decision or verification target, not because it happens to be available.
