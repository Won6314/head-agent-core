# When You Need HEAD

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [Adoption](README.md) / When You Need HEAD

## Learning Objective

Recognize when an explicit outcome owner improves work and when a direct conversation or ordinary workflow is enough.

## Core Claim

Use HEAD when preserving the whole outcome, resolving dependencies, or verifying a result is harder than performing one clear action. Do not add it merely because an LLM is available.

| Work shape | Useful response |
| --- | --- |
| One clear, low-consequence action with an immediate check | Work directly; no run or worker is implied. |
| Work with multiple dependencies, handoffs, or an interruption risk | Let HEAD make the work model and preserve the agreement. |
| A bounded result that benefits from separate execution | Delegate one coherent outcome, then verify it. |
| A consequential result where a second judgment could change the decision | Add independent review. |
| A material product, policy, architecture, cost, risk, or workflow choice | Keep the human decision gate explicit. |

## Unsuitable Uses

HEAD is not a reason to turn a simple edit into a planning exercise, to substitute generated confidence for evidence, or to conceal an unresolved human decision behind an agent workflow. A swarm is also a poor default: more owners create more context, coordination, and reconciliation work.

## A Practical Test

Ask four questions:

1. What result must be true at the end?
2. What could be forgotten, guessed, or changed if the work continues later?
3. Does a separate owner have a coherent result and observable completion evidence?
4. Which choice would materially alter the outcome and therefore remains human-owned?

If the answers are simple and stable, direct work is usually preferable. If they reveal continuity, coordination, or evidence problems, add only the layer that addresses that problem.

## Common Misunderstanding

"More structure means more reliability." Structure can make decisions and evidence easier to inspect, but it cannot make missing facts true or poor judgment correct.

## Takeaway

Adopt HEAD for complexity that needs ownership and verification, not as a default ritual.
