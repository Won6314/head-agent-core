# Minimal Adoption

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [Adoption](README.md) / Minimal Adoption

## Learning Objective

Start with the smallest composition that preserves the needed ownership boundary without importing a complete operating system.

## Core Claim

The minimum viable model has a shared reasoning layer, a project-owned layer, and a human-owned direction. Add a session canon for durable work and a worker path only for a genuinely bounded outcome.

```text
Human: fixes goal and material choices
        |
        v
Shared Core: portable ownership and reasoning principles
        +
Project layer: local authority, evidence index, and constraints
        |
        v
HEAD: models, executes or delegates, verifies, and concludes
```

## Start Small

1. Use a compact shared Core that says HEAD owns the whole outcome, composes authoritative context deliberately, preserves coherent ownership, and verifies results.
2. Create project-owned instructions that define local authority, constraints, sources of truth, and approval boundaries. Keep a small index that points to detailed evidence instead of copying it into every conversation.
3. Let HEAD work directly for small tasks. Record a durable session canon only when coordination, interruption, recovery, or handoff makes it useful.
4. For durable work, keep stable session identity separate from the full user-HEAD agreement. The agreement retains requirements, scope, success conditions, work model, checklist, current position, assumptions, next action, and evidence references. A summary may help retrieval but cannot replace the agreement.
5. Add an optional worker path only when one owner can carry a clear result end to end. Give that owner the smallest complete context, require direct outcome evidence, and have HEAD integrate the verified result.

## What Not To Add Yet

Do not begin with a large roster, specialized integrations, a mandatory template, or a database of task state. Those may solve real later problems, but they also create maintenance and authority surfaces that the initial work may not need.

## Design Response

The shared [Core](../../head/README.md) establishes portable behavior. The [Project Layer](../../projects/README.md) adds local meaning. A run-oriented procedure is available through [Shared Skills](../../skills/README.md); coordination interfaces and reusable roles are optional [MCP](../../mcp/README.md) and [Agent](../../agents/README.md) layers.

## Common Misunderstanding

"Minimal" does not mean "underspecified." A small system still needs a clear result, an authority boundary, and a way to observe completion.

## Takeaway

Begin with human direction, shared principles, project authority, and direct verification. Add durability and delegation only when the work demands them.
