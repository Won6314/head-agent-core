# Limitations

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [Adoption](README.md) / Limitations

## Learning Objective

Adopt the model with clear expectations about what it can improve and what remains uncertain, costly, or human-owned.

## Core Claim

HEAD can make work more inspectable and recoverable; it cannot guarantee truth, correct use of tools, correct authority, or good human decisions.

| Limitation | Consequence |
| --- | --- |
| HEAD can be wrong | A coherent work model can rest on incomplete or mistaken evidence. Recheck mutable facts and invite challenge where it matters. |
| Delegation has cost | Briefing, waiting, verification, and integration can be slower than direct work. |
| Incorrect authority can scale | A bad decision boundary can propagate a mistake consistently across workers and tools. |
| Tool availability does not ensure appropriate use | A callable capability may be ignored, misused, unavailable, or insufficient for the real-world condition. |
| Canon preserves agreement, not reality | A durable record helps prevent scope drift but must not be mistaken for fresh external evidence. |
| Independent review is bounded | A reviewer can miss errors, share the same blind spots, or lack the evidence needed to challenge a result. |
| User direction remains necessary | Material choices about product, policy, architecture, cost, risk, workflow, and public communication cannot be delegated away by process design. |

## Design Response

Use direct evidence, preserve the user-HEAD agreement for durable work, and add independent review when it can materially change a consequential result. When evidence is absent or a decision crosses the human boundary, state the uncertainty rather than filling it with plausible invention.

## Related Theory

The model resembles a control plane coordinating bounded execution, but the analogy has limits. Software architecture terms do not make an LLM system deterministic, complete, or self-governing.

## Takeaway

The value of HEAD is disciplined coordination under uncertainty, not automation that replaces human responsibility.
