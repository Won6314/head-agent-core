# Common Antipatterns

[HEAD Agent Core](../../README.md) / [Learn](../README.md) / [Adoption](README.md) / Common Antipatterns

## Learning Objective

Recognize additions that appear organized but weaken ownership, evidence, or adaptability.

## Core Claim

Most adoption failures come from treating mechanism as authority: process replaces judgment, context replaces evidence, or a worker result replaces the human-HEAD agreement.

| Antipattern | Why it fails | Better response |
| --- | --- | --- |
| Ceremony for every task | Planning artifacts consume attention without changing a small result | Scale the process to coordination, recovery, and risk. |
| Context hoarding | Volume introduces stale, conflicting, and irrelevant material | Retrieve the smallest authoritative set that changes the decision. |
| Worker swarm | Many partial owners create integration gaps and incompatible assumptions | Assign one coherent outcome to one owner; parallelize only independent results. |
| Hidden authority | A worker or tool silently makes a material product or policy choice | Surface the decision and return it to the human owner. |
| Summary as canon | A lossy handoff can omit scope, constraints, or unfinished work | Keep stable identity and the full agreement authoritative outside summaries. |
| Tool-first design | An available interface is mistaken for a justified workflow | Define the outcome and safety boundary before choosing a tool. |
| Checklist completion as proof | Completed actions can still miss the requested behavior | Require direct evidence of the outcome. |
| Copying project guidance into Core | Local assumptions become false universal rules | Keep project facts and procedures in project-owned layers. |

## Generalized Failure

A team can add workers after observing slow progress, then add summaries to coordinate them, then add more control mechanisms to repair summary drift. The visible system grows while the original outcome becomes harder to see. The corrective move is not another mechanism by default: restore the authoritative agreement, reduce ownership overlap, and retain only the layer that addresses an observed need.

## Common Misunderstanding

"Autonomy means agents decide everything." Useful autonomy is technical freedom inside a bounded outcome. It does not include inventing policy, product direction, or authority.

## Takeaway

Prefer explicit ownership, direct evidence, and the smallest adequate mechanism over an impressive-looking process.
