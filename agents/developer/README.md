## Why It Exists

Implementation work benefits from one owner carrying a coherent result from diagnosis through direct evidence. A shared contract keeps that ownership model consistent across repositories and technology stacks.

## Boundary

The shared role defines implementation authority and return expectations. Repository ownership, product policy, local commands, and project communication channels belong in the project's Developer overlay.
