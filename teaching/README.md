## Facilitation Boundary

Use the routes as a guide, not as permission to invent policy. Keep these distinctions explicit:

- The User fixes direction and material decisions.
- HEAD owns the work model, evidence selection, integration, and ordinary planning.
- A bounded Agent owns local execution within a supplied boundary and returns direct completion evidence.
- Generated output remains a draft until checked against evidence appropriate to its claim.

The course's Historical record, Operational observation, Generalized failure, and Related theory labels retain their meanings in a teaching setting. Do not present retrospective theory as the original design history, or use public examples to infer private project details.

## Delivery Pattern

1. State the learning objective and the session boundary.
2. Teach from the linked learning pages; use each diagram from [Canonical diagrams](diagrams.md).
3. Ask participants to distinguish observed evidence from plausible output.
4. Close by having participants name one decision owner, one evidence gate, and one next bounded outcome.

For a first session, start with the [20-minute introduction](20-minute-introduction.md). For a complete conceptual sequence, use the [60-minute core lecture](60-minute-core-lecture.md).
