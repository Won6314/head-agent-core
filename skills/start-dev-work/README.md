## Why It Exists

Software work needs an explicit model of the existing system, intended behavior, interfaces, ownership, and execution order before implementation begins. This skill adds that architecture-specific preparation without burdening non-development work.

## Boundary

The skill defines how HEAD prepares a composable implementation slice. Project discovery tools, repositories, architecture decisions, and coding standards remain project-owned.
